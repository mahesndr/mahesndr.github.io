What is GNU/Linux?

Bismillahirrahmanirrahim.

This article tries to explain GNU/Linux accurately, in accordance with its measurement, concise, and easy to understand. This article was made because of dissatisfaction with the writings on the same topic in Indonesian that are available on the internet. Because there is confusion that results in reader confusion, especially one common mistake of assuming “Everything was made by Linus Torvalds in 1991 with a little help from the GNU Project”. This paper was written to clean up the error and provide the correct facts about GNU / Linux. Hopefully this article is useful.

What is GNU

GNU is a free operating system that is like UNIX®. GNU is UNIX® which was remade from stratch as free software. GNU is a UNIX® clone. The GNU project aims to produce a complete operating system as a whole, not to focus on just one part of it. GNU was developed from scratch which means there is no UNIX® source code in GNU. The GNU operating system is a complete free software system, upward-compatible with Unix. GNU stands for “GNU’s Not Unix”. It is pronounced as one syllable with a hard g. Richard Stallman made the Initial Announcement of the GNU Project in September 1983.

What is Linux

Linux is a free UNIX-like kernel. Linux is a MINIX clone (a UNIX-like operating system) as a kernel component. The Linux kernel was developed from scratch, meaning there is no MINIX or UNIX® source code on Linux. The Linux kernel is compatible with both UNIX® and UNIX-like operating systems, because it adheres to the UNIX® standard called POSIX. The Linux kernel has been worked on by Linus Torvalds since 1991 with the community. The kernel name “Linux” comes from the name Linus as its discoverer with the meaning of Linus’ UNIX (Linus’s UNIX). The Linux kernel is generally included in the GNU operating system and is redistributed as a variety of stand-alone operating systems. Something like this is called the GNU / Linux distribution. In addition, there is also a Linux kernel that is distributed in operating systems other than GNU, for example on Android. Something like this can be called a Linux distribution.

What is GNU / Linux

GNU / Linux is a name for the GNU operating system that uses the Linux kernel. GNU / Linux is the name for each operating system which is a composition of GNU and the Linux kernel. GNU / Linux is the name for all operating system distributions such as Slackware, Debian, SUSE Enterprise Linux, Red Hat Enterprise Linux, Gentoo, Mandriva, Arch Linux, and so on. The meaning of the name GNU / Linux is to show the operating system called GNU with a kernel component called Linux. Naming the GNU / Linux is parallel with another operating system called GNU / Hurd, the GNU operating system with the Hurd kernel. The GNU / Linux site is very large according to the number of distributions available. For example the Slackware site https://slackware.com and the Ubuntu site https://ubuntu.com.

What is the GNU / Linux Platform?

The GNU / Linux platform is a designation for the total hardware, software, and operating system around GNU / Linux. The GNU / Linux platform can be called simply the GNU Platform. The GNU / Linux or GNU platforms parallel to other platforms can be sequenced in a series of platforms: UNIX®, BSD, GNU, Windows, Mac. Each platform in the series has its own hardware support, also has its own software support. Therefore the software available for Windows is not the same as Mac, on the contrary the software for GNU / Linux is not the same as Windows or BSD. Likewise hardware.

The GNU / Linux or GNU platform covers the entire operating system distribution of GNU / Linux. Each GNU / Linux operating system has its own platform. For example Slackware, has become an operating system as well as an independent platform as well. Slackware as GNU / Linux is already comparable to Windows and Mac. Similarly, GNU / Linux in addition to Slackware, such as Debian, Fedora, and so forth. Compared to the Windows or Mac platforms, the platforms in the GNU platform if the total area is much larger.

In some ways GNU is the same as UNIX, including derivatives. Therefore GNU has many derivatives (called distributions) operating systems (such as Slackware to Linux Mint) as UNIX® also has UNIX® derivatives (such as IBM AIX to Sun Solaris).

How GNU / Linux was developed

The free software community does not have the concept of vendorisation, that is, one product is made by only one manufacturer (like Windows). In the world of free software, the concept is upstream and downstream. Upstream is (hulu:indonesian), that is, every original developer of a software. Downstream is (hilir:indonesian), ie every developer outside the upstream who redistributes software from upstream. GNU was developed by the GNU Project, producing thousands of components and actively developing it, and in this case the GNU Project is upstream in the free software community. The GNU project in this case is a factory that produces all major components of its operating system. Other upstream in the community are Project X11, KDE Project, Linux Kernel Project, and so on. All of that is upstream, which is the community that produces components or software.

Outside the upstream, there are other projects such as Slackware, Debian, SUSE Linux Enterprise, Red Hat Enterprise Linux, and others. These projects are downstream. They take software from upstream then process it and become GNU / Linux distributions. Every GNU / Linux distribution (eg Slackware) is the perfect operating system. These downstream projects make the distribution (or distribution) of GNU / Linux such as Ubuntu or Trisquel used by end users. Every single GNU / Linux distribution is an operating system equivalent to Windows or Mac OS X. This is an outline of how a GNU / Linux operating system is realized.

What is free software

The free software definition presents the criteria for whether a particular software program qualifies as free software. “Free software” means software that respects users’ freedom and community. Roughly, it means that the users have the freedom to run, copy, distribute, study, change and improve the software. Thus, “free software” is a matter of liberty, not price. To understand the concept, you should think of “free” as in “free speech,” not as in “free beer”. We sometimes call it “libre software,” borrowing the French or Spanish word for “free” as in freedom, to show we do not mean the software is gratis.

“Free software” does not mean “noncommercial”. A free program must be available for commercial use, commercial development, and commercial distribution. Commercial development of free software is no longer unusual; such free commercial software is very important. You may have paid money to get copies of free software, or you may have obtained copies at no charge. But regardless of how you got your copies, you always have the freedom to copy and change the software, even to sell copies.
